# Dashboard Editor 开发文档

> 适用场景：Vue 3 中后台管理系统、监控仪表盘、数据可视化编辑器
> 读者：开发人员 + AI Agent

---

## 一、技术栈概览

| 层级 | 技术选型 | 版本 |
|------|---------|------|
| 前端框架 | Vue 3 | ^3.5 |
| 构建工具 | Vite | ^6.3 |
| UI 组件库 | Ant Design Vue | ^4.2.6（全量全局注册） |
| 图表库 | AntV G2 | ^5.x |
| 图可视化 | AntV G6 | ^5.x |
| 图标库 | Font Awesome 6 | CDN 引入（`fa-regular` / `fa-solid`） |
| 后端 | Express | Node.js |
| 数据库 | PostgreSQL | 192.168.0.155:2345 |

## 三、Plan 模式开发流程

开发前执行"需求理解 → 多轮验证 → 确认执行"，避免做错需求。

### 3.1 需求理解

```
用户输入需求
  ↓
列出理解要点（确认清单）
  ↓
逐条与用户验证
  ↓
若有歧义 → 呈现多个选项让用户选择
  ↓
确认后开始执行
```

### 3.2 执行流程

1. **拆分步骤**：将任务拆成可验证的步骤
2. **每步验证**：每一步完成后立即验证（如：页面渲染、API 响应、测试通过）
3. **循环调整**：验证失败 → 定位修复 → 重新验证
4. **完整闭环**：全部步骤完成 → 整体回归验证

### 3.3 输出确认清单

```markdown
1. [Step] → verify: [check]
2. [Step] → verify: [check]
3. [Step] → verify: [check]
```

### 3.4 防止常见误解

- 不确定时**必须提问**，不要猜
- 多个可行方案时先列出来让用户选
- 如果发现更简单的方法，主动提出

## 四、Karpathy 开发准则

### 4.1 Think Before Coding（先想再做）

实施前明确假设，不确定就问，有歧义先对齐。

```javascript
// ❌ 直接开始写，遇到问题再改
function renderChart() { ... }

// ✅ 先确认：数据格式？图表类型？容器是否存在？
// 再编码
```

### 4.2 Simplicity First（能简则简）

- 不多写一行不需要的代码
- 不要为了"灵活性"加未需求的功能
- 不创建单一用途的抽象
- 不自以为是的"优化"
- 200 行能写成 50 行 → 重写

自问："高级工程师会觉得这个过于复杂吗？"

### 4.3 Surgical Changes（精准修改）

- 只改必须改的代码
- 不改相邻代码、注释、格式
- 不改不相关的东西
- 匹配现有代码风格（即使你偏好不同）
- 发现无关的死代码 → 提一下，但不要删

**每行改动必须能追溯到用户需求。**

### 4.4 Goal-Driven（目标驱动）

把任务转成可验证的目标：

```
"添加验证" → "写测试覆盖无效输入，然后让测试通过"
"修 Bug"   → "写测试复现，然后让测试通过"
"重构 X"   → "确保重构前后测试都通过"
```

### 4.5 修改后清理

- 删掉你改代码后不再用的 import、变量、函数
- 不要删改前就存在的死代码（除非用户要求）

### 5.1 安装与配置

**注意**：Vite 配置了 basic auth（admin/745544752），URL 必须用 `http://user:pass@host/` 格式。这会触发 Vue Router `SecurityError: replaceState`（不影响渲染）。

### 5.2 开发闭环

```
写功能代码 → 写测试 → 运行测试 → 测试通过 → 下一功能
                ↑ 失败 → 修复 → 重跑
```

### 5.3 测试编写规范

```javascript
import { test, expect } from '@playwright/test'

test.describe('模块名', () => {
  test('具体场景描述', async ({ page }) => {
    await page.goto('/target-path')
    await expect(page.locator('#app')).toBeAttached()
    // 断言具体内容
  })
})
```

### 5.4 G2 图表验证注意

G2 v5 默认渲染到 **Canvas** 而非 SVG。验证图表时：
- 不能用 `querySelectorAll('svg')` 检查渲染
- 通过 `canvas.getContext('2d').getImageData()` 或 `canvas.width` 验证
- 优先用 playwright `eval` + 文本 snapshot，而非截图后 OCR

## 六、状态管理

### 6.1 单例模式（useEditorState）

```javascript
// src/composables/useEditorState.js
const state = reactive({
  charts: [],
  selectedId: null,
  configOpen: false,
})

export function useEditorState() {
  return { state, /* ...方法 */ }
}
```

### 6.2 正确使用

```javascript
// ✅ 正确 - 共享同一个响应式对象
const { state } = useEditorState()
state.charts.push(newChart)

// ❌ 错误 - 每次返回新对象
export function useEditorState() {
  return reactive({ charts: [] })
}

// ❌ 错误 - 非响应式解构
const { charts } = state  // 失去响应式
const { charts } = toRefs(state)  // ✅ 保持响应式
```

### 6.3 数组修改

```javascript
// ✅ 通过索引修改
state.charts[index].title = '新标题'
// ✅ push
state.charts.push(newChart)
// ❌ 直接替换数组
state.charts = newArray
```

## 七、组件规范

### 7.1 列表页模板

```
.page-header > h3
.filter-bar (a-select, a-range-picker, a-input-search, button)
a-table
```

- 搜索框在 filter-bar 最右侧，新建按钮 `margin-left: auto`
- 控件默认 32px（不加 `size="small"`）
- 间距：header→bar=16px, bar→table=16px, gap=8px
- 三层布局：第1行按钮、第2行筛选（搜索框 `flex: 1`）、第3行表格

### 7.2 Props / Emits

```javascript
const props = defineProps({
  type: { type: String, default: 'line' },
})
const emit = defineEmits(['toggle', 'remove'])
```

### 7.3 template 注意

- `a-table` `:row-class-name` 签名 `(record, index)`，非 `({ record })`
- **避免 template 箭头函数绑定** — AntD 回调用 `function() {}`

### 7.4 Ant Design column filter 优先

列表页状态筛选优先使用 `a-table` column 的 `filters` + `onFilter`，而非在 filter-bar 额外放 Select：

```javascript
{
  title: '状态',
  dataIndex: 'status',
  filters: [{ text: '已启用', value: 'enabled' }],
  onFilter: (value, record) => record.status === value,
}
```

### 7.5 Modal 标题对齐

```vue
<a-modal>
  <template #title>标题</template>  <!-- 用 slot 替代 title prop -->
</a-modal>
```

对应 CSS：`:deep(.ant-modal-header) { padding: 16px 24px }`

## 九、图表开发经验

### 9.1 AntV G2 渲染

```javascript
import { Chart } from '@antv/g2'

function renderChart() {
  if (chart) { chart.destroy(); chart = null }
  if (!container.value) return
  chart = new Chart({ container: container.value, autoFit: true, padding: [8, 8, 16, 36] })

  chart.line().data(useData).encode('x', 'time').encode('y', 'value')
  chart.render()  // async
}
```

### 9.2 G2 要点

- `chart.render()` 是 async
- **柱状图数据必须放在 interval mark 层级**：`chart.interval().data([...])`，而非 `chart.data([...])` + `chart.interval()`。折线图/面积图用 mark-level data 无此问题
- G2 v5 默认渲染到 Canvas，不是 SVG
- 调用不存在的函数名（如 `renderTopnChart` vs `renderTopNChart`）会静默抛 TypeError

### 9.3 G2 destroy 防泄漏

```javascript
watch(() => [props.type, props.color, props.data],
  () => nextTick(() => renderChart()), { deep: true })

onBeforeUnmount(() => {
  if (chart) { chart.destroy(); chart = null }
})
```

### 9.4 小卡片用 a-progress 替代 G2

G2 环形图/柱状图在小卡片中空间利用率低且依赖 Canvas 生命周期。改用 Ant Design Vue `a-progress` 更紧凑稳定：

```vue
<a-progress :percent="usage" type="dashboard" />
```

### 9.5 SVG 数值卡片

```javascript
function renderNumericSVG(color, val) {
  return `<svg viewBox="0 0 340 165">...</svg>`
}
```

### 9.6 G6 v5 注意

- 无 `sourceAnchor`，用 port 系统
- `getChildrenData(comboId)` 返回空 → 改用 `getNodeData().filter(d => d.combo === id)`

### 9.7 图表推荐逻辑

```javascript
const recommendedCharts = computed(() => {
  // period='now' + 单资源 → numeric/line
  // period='now' + 多资源 → bar/numeric
  // 时间段 → line/area
})
```

### 9.8 容器高度为 0 兜底

```javascript
if (container.value.clientWidth > 0 && container.value.clientHeight === 0) {
  container.value.style.minHeight = '155px'
}
```

## 十、项目踩坑合集

### API 与后端

| 问题 | 原因 | 解法 |
|------|------|------|
| Express 路由捕获 `:tableName` | cmdbRouter catch-all 中间件拦截 | 自定义端点放独立路径（`/api/vm/*`），在 `app.use('/api/cmdb')` 前注册 |
| Vite proxy 404 | 后端新增端点后没加代理 | 同步更新 `vite.config.js` proxy 配置 |
| 后端 JSON 数组字段前端报错 | 两次 `JSON.parse()`（`res.json()` + `fetch().json()`） | 直接访问数组/对象属性，不要 `JSON.parse()` |
| AI Chat 空回复 | Agnes API 偶发返回 `content=""` | for-loop 重试（最多 2 次，间隔 1s） |
| Node.js fetch 静默失败 | undici TLS 兼容性问题 | 用 `exec(curl)` 回退 |

### AI Assistant

| 问题 | 解法 |
|------|------|
| AI 回复 `reply=""` 但 `actions` 存在 | 判空用 `if (data.reply \|\| data.actions?.length)` |
| 推荐按钮格式 | `[[action:按钮文字:补充prompt]]`，后端正则 `/(\[action:([^\]]+)\])/g` 解析 |

### 构建与部署

| 问题 | 解法 |
|------|------|
| `nohup` 启动找不到命令 | 写死全路径或 `export PATH="/usr/local/bin:/opt/homebrew/bin:$PATH"` |
| Bash 工具超时杀死前台进程 | 必须用 `nohup` 启动脱离进程组，或用 `start.sh` |
| HMR 静默失效 | 刷新页面即可 |
| basic auth + `SecurityError: replaceState` | 不影响渲染，忽略 |

### Dashboard

| 问题 | 解法 |
|------|------|
| OBSDashboard 特殊页签 | 判断 `currentDashboard.value?.title === 'OBS监控'`，隐藏通用 toolbar/FAB |
| 自定义 Dashboard toolbar | 自包含刷新、时间范围等交互，不依赖 App.vue 通用 toolbar |
| Dashboard slug 路由 | 用 `/dashboard/vm` 而非 `/dashboard/6` |
| G2 图表 OCR 识别差 | 优先用 playwright `eval` + 文本 snapshot |

### 样式

| 问题 | 解法 |
|------|------|
| 嵌套路由 padding 叠加 | 父容器有 padding 时子页面不要重复设 |
| Toast 与 FAB 重叠 | Toast 留出 FAB 空间（bottom: 96px） |
| 选中态 box-shadow 不可见 | 用品牌色 3px solid，不要用浅色 shadow |
| fixed 元素在 overflow 父容器内被裁剪 | 挂 body 层级 |

## 十一、常见问题排查

### G2 图表不渲染

1. 确认 `chart.render()` 已调用（async）
2. 柱状图数据是否放在 interval mark 层级
3. 检查 chartType 与数据匹配
4. 容器高度是否 > 0

### API 请求失败

1. 检查 Vite proxy 配置是否添加了新路径
2. 后端是否已启动（`lsof -i :3001`）
3. Express 路由顺序是否正确（catch-all 放最后）

### 响应式数据不更新

1. 是否用了非响应式解构（`const { a } = state`）
2. 是否直接替换了数组（`state.charts = newArray`）
3. 是否使用了非响应式对象

### iOS Safari 兼容

- 使用 `100dvh` 替代 `100vh`
- 慎用 `position: sticky`
- 使用 `-webkit-` 前缀

### 调试

- basic auth 验证：`curl -u admin:745544752 http://localhost:5173`
- 端口确认：`lsof -i :5173 -i :3001`
- 服务状态：`ps aux | grep -E "node|vite"`
