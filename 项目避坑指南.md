# 项目避坑指南

本文档总结仪表盘编辑器项目开发中积累的经验和教训，供其他项目参考借鉴。

> **适用场景**：Vue 3 中后台管理系统、监控仪表盘、数据可视化编辑器
> **技术栈**：Vue 3 + Vite + Ant Design Vue 4.x + AntV G2

---

## 一、项目初始化与架构

### 1. Vite 项目创建

```bash
npm create vite@latest dashboard-editor -- --template vue
cd dashboard-editor
npm install
npm install ant-design-vue@4 @antv/g2 @fortawesome/fontawesome-free
```

**经验**：
- 使用 `npm create vite` 比手动配置更规范，避免版本兼容问题
- 依赖版本锁定在 `package.json` 中指定，如 `"ant-design-vue": "^4.2.6"`

### 2. 目录结构设计

```
src/
├── main.js                    # Vue 应用入口
├── App.vue                    # 主布局 + 全局 CSS 变量
├── composables/
│   ├── useEditorState.js      # 全部状态 + 业务逻辑 (核心)
│   └── usePrometheus.js       # API 数据获取
└── components/
    ├── ChartGrid.vue          # 图表网格 (v-for + 添加卡片)
    ├── ConfigPanel.vue        # 配置面板 (数据/样式 Tab)
    ├── ChartRenderer.vue      # 图表渲染 (SVG/G2)
    └── MultiSelect.vue        # 通用多选下拉
```

**经验**：
- 业务逻辑集中在一个 `composable` 中管理，避免状态分散
- 组件保持单一职责，ChartGrid 只负责网格布局

### 3. Vite 配置 (vite.config.js)

```javascript

```

**经验**：
- 开发环境需要配置 API 代理解决跨域
- `allowedHosts` 用于支持内网穿透访问

## 二、状态管理 (Composition API)

### 1. 单例模式状态管理

```javascript
// src/composables/useEditorState.js

// 定义状态 (只创建一次)
const state = reactive({
  charts: [],
  selectedId: null,
  configOpen: false,
  // ...其他状态
})

// 导出 composable 函数
export function useEditorState() {
  return {
    state,
    // ...方法
  }
}
```

**✅ 正确写法**：
```javascript
// 组件中使用 - 共享同一个响应式对象
import { useEditorState } from './composables/useEditorState'
const { state } = useEditorState()
state.charts.push(newChart)  // 响应式更新
```

**❌ 错误写法**：
```javascript
// 每次调用创建新实例
export function useEditorState() {
  return reactive({ charts: [] })  // ❌ 每次返回新对象，不共享状态
}
```

### 2. 计算属性 (computed)

```javascript
// 当前选中的图表
const currentChart = computed(() =>
  state.charts.find(ch => ch.id === state.selectedId) || null
)

// 级联选择的当前分类
const currentCategory = computed(() =>
  CASCADE[state.dsType]?.[state.dsValue[0]] || null
)

// 推荐图表列表
const recommendedCharts = computed(() => {
  if (!state.selectedMetrics.length) return { items: [], types: [], cur: '' }
  // ...推荐逻辑
  return { items: sorted, types, cur, mode: modeLabel, warning }
})
```

### 3. 响应式数据注意事项

```javascript
// ✅ 正确 - 通过索引修改数组元素
state.charts[index].title = '新标题'

// ✅ 正确 - 使用 push 修改数组
state.charts.push(newChart)

// ✅ 正确 - 直接修改对象属性
state.selectedId = 123

// ❌ 错误 - 直接替换整个数组（可能不触发更新）
state.charts = newArray

// ❌ 错误 - 使用非响应式的解构
const { charts } = state  // ❌ 失去响应式
const { charts } = toRefs(state)  // ✅ 保持响应式
```

### 4. 状态重置模式

```javascript
// 新建图表时重置状态
function addChart() {
  state.configOpen = true
  state.configTab = 'data'
  state.selectedId = null  // 新建模式标志
  state.selectedMetrics = []
  state.dsValue = []
  state.period = undefined  // 未选择状态
  state.objType = 'all'
}
```

## 三、移动端开发避坑

### 1. 视口高度单位 (最重要)

| 单位 | 说明 | 移动端表现 |
|------|------|----------|
| `100vh` | 视口高度 | ❌ 包含浏览器工具栏，导致计算偏差 |
| `100dvh` | 动态视口高度 | ✅ 自动适配工具栏 (推荐) |
| `100svh` | 最小动态视口高度 | ✅ 兼容性好 |

```css
/* 正确 */
.canvas {
  height: calc(100dvh - 48px);  /* 减去 header 高度 */
}

/* 错误 */
.canvas {
  height: 100vh;  /* ❌ 移动端会出现滚动问题 */
}
```

### 2. 弹层面板定位

```css
/* 正确 */
.config-panel {
  position: fixed;  /* ✅ 相对于视口定位 */
  top: 0;
  right: 0;
  bottom: 0;
  width: 420px;
}

/* 错误 */
.config-panel {
  position: absolute;  /* ❌ 相对于父容器定位，复杂布局中易出问题 */
}
```

### 3. 移动端响应式断点

```css
/* 平板 */
@media (max-width: 1024px) {
  .config-panel { width: 360px; }
}

/* 手机竖屏 */
@media (max-width: 768px) {
  .config-panel {
    width: 100%;
    height: 100%;
    top: 0;  /* 移动端必须 top:0，底部按钮才能固定 */
  }
  .config-footer {
    flex-shrink: 0;  /* 防止按钮被压缩 */
  }
}

/* 小屏手机 */
@media (max-width: 420px) {
  .config-scroll { padding: 12px; }
}
```

### 4. 背景滚动锁定

```css
/* 锁定背景滚动 */
body.noscroll {
  overflow: hidden;
  position: fixed;
  width: 100%;
  height: 100%;
}
```

```javascript

const isMobile = ref(window.innerWidth <= 768)

watch(() => state.configOpen, (open) => {
  if (open && isMobile.value) {
    document.body.classList.add('noscroll')

    document.body.classList.remove('noscroll')

```

### 5. 触摸滚动优化

```css
/* 确保触摸事件正确传递 */
.scrollable {
  touch-action: pan-y;  /* 只允许垂直滚动 */
  -webkit-overflow-scrolling: touch;  /* iOS 惯性滚动 */
}

/* overlay 阻挡了底层滚动时，让触摸穿透 */

  pointer-events: none;
}
```

### 6. 移动端检测与防抖

```javascript
import { ref, onMounted, onUnmounted } from 'vue'

const isMobile = ref(window.innerWidth <= 768)
let resizeTimer = null

onMounted(() => {
  window.addEventListener('resize', () => {
    clearTimeout(resizeTimer)
    resizeTimer = setTimeout(() => {
      isMobile.value = window.innerWidth <= 768
    }, 100)  // 100ms 防抖
  })
})

onUnmounted(() => {
  window.removeEventListener('resize', () => {})
})
```

## 四、Vue 3 模板语法迁移

### 1. 从 jQuery/Vue 2 迁移

| jQuery / Vue 2 | Vue 3 Composition |
|---------------|-------------------|
| `getElementById('id')` | `v-model` 双向绑定 |
| `innerHTML` | `v-html` |
| `onclick` | `@click` |
| `getAttribute('data-xxx')` | 传递 props 或用 `:data-xxx` |
| `style.display` | `v-show` / `v-if` |

```html
<!-- Vue 2 / 原生 -->
<div id="chart-1" onclick="handleClick(this)" data-id="1"></div>

<!-- Vue 3 -->
<div
  :data-chart-id="chart.id"
  @click="handleClick(chart.id)"
  :class="{ selected: state.selectedId === chart.id }"
></div>
```

### 2. 条件渲染 vs 展示/隐藏

```html
<!-- v-if: 不渲染 DOM -->
<div v-if="state.configOpen" class="panel"></div>

<!-- v-show: 渲染但隐藏 (不适合复杂动画) -->
<div v-show="state.configOpen" class="panel"></div>

<!-- 动画过渡 -->
<transition name="fade">
  <div v-if="state.configOpen" class="panel"></div>
</transition>
```

### 3. 列表渲染与 Key

```html
<!-- 必须加 key -->
<div v-for="ch in state.charts" :key="ch.id" class="chart-card">
  ...
</div>

<!-- 不推荐用 index 作为 key -->
<div v-for="(ch, index) in state.charts" :key="index">  // ❌
```

## 六、图表开发经验

### 1. SVG 图表渲染

```javascript
// ChartRenderer.vue
function renderNumericSVG(c, val) {
  const displayVal = val != null ? Math.round(val * 10) / 10 + '%' : '2,847'
  const trend = val != null ? (val >= 0 ? '+12.4%' : '-3.2%') : '+12.4%'

  return `<svg viewBox="0 0 340 165" ...>
    <text x="170" y="72" ...>${displayVal}</text>
    ...
  </svg>`
}
```

```html
<!-- 模板中使用 v-html -->
<div v-if="type === 'numeric'" v-html="renderNumericSVG(color, realValue)"></div>
```

### 2. AntV G2 图表配置

```javascript
import { Chart } from '@antv/g2'

function renderChart() {
  if (chart) { chart.destroy(); chart = null }
  if (!container.value || props.type === 'numeric') return

  chart = new Chart({
    container: container.value,
    autoFit: true,
    padding: [8, 8, 16, 36],
  })

  if (props.type === 'line') {
    chart.area()
      .data(useData)
      .encode('x', 'time').encode('y', 'value')
      .style('fill', `l(0) 0:${c}20 1:${c}00`)
      .style('shape', 'smooth')
    chart.line()
      .data(useData)
      .encode('x', 'time').encode('y', 'value')
      .style('stroke', c).style('lineWidth', 2)
  }

  chart.axis('x', { labelFontSize: 9, labelFill: '#9CA3AF', title: null })
  chart.axis('y', { labelFontSize: 9, labelFill: '#9CA3AF', gridStroke: '#E5E5EA' })
  chart.render()
}
```

### 3. 图表类型切换 (watch)

```javascript
import { watch, nextTick } from 'vue'

watch(
  () => [props.type, props.color, props.data],
  () => nextTick(() => renderChart()),
  { deep: true }
)

onBeforeUnmount(() => {
  if (chart) { chart.destroy(); chart = null }
})
```

### 4. 图表推荐逻辑

```javascript
const recommendedCharts = computed(() => {
  if (!state.selectedMetrics.length) return { items: [], types: [], cur: '' }

  const isNow = state.period === 'now'
  const isSingleResource = state.objType === 'spec' && state.selectedResources.length === 1

  let types = []
  if (isNow) {
    if (isSingleResource) {
      types = ['numeric', 'line']  // 当前值 + 单资源 → 数值卡片
    } else {
      types = ['bar', 'numeric']   // 当前值 + 多资源 → 柱状图
    }
  } else {
    types = ['line', 'area']       // 时间段 → 折线图/面积图
  }

  // 只显示推荐的图表类型 (过滤掉不推荐的)
  const filtered = CHART_DEFS.filter(c => types.includes(c.t))

  return { items: sorted, types, cur: types[0], mode: modeLabel, warning }
})
```

## 九、常见问题排查

### 1. 样式不生效

- 检查是否使用了 `<style scoped>`，scoped 样式需要带 data 属性
- 检查 CSS 选择器优先级
- 检查是否有组件层级覆盖问题
- 尝试加 `!important` 确认是否是优先级问题

### 2. 响应式数据不更新

```javascript
// 检查是否直接修改了对象属性而不是响应式对象
const obj = reactive({ a: 1 })
obj.a = 2  // ✅ 正确

const plain = { a: 1 }
plain.a = 2  // ❌ 非响应式

// 检查是否使用了非响应式的解构
const { a } = state  // ❌ 失去响应式
const { a } = toRefs(state)  // ✅ 保持响应式
```

### 3. 移动端点击延迟

```css
/* 移除点击延迟 */
* {
  touch-action: manipulation;
}
```

### 4. iOS Safari 兼容性问题

- 避免使用 `100vh`，使用 `100dvh` 或 `100svh`
- 使用 `-webkit-` 前缀兼容 CSS 属性
- 慎用 `position: sticky`，在 iOS 有时表现异常

### 5. Logo 加载失败

```vue
<!-- 加 fallback -->
<img
  :src="logoUrl"
  @error="$event.target.src = '/logo-fallback.svg'"
  alt="Logo"
  class="logo-img"
>
```

### 6. 图表容器高度为 0

```javascript
// ChartRenderer.vue
if (container.value.clientWidth > 0 && container.value.clientHeight === 0) {
  container.value.style.minHeight = '155px'
}
```

## 十、开发流程建议

### 1. 先文档后代码

```
需求文档 → 技术方案 → 代码实现 → 验证对照
```

复杂逻辑修改前，先更新文档确保逻辑一致。

### 2. 移动端优先

```css
/* 默认移动端样式 */
.card { padding: 12px; }

/* 桌面端扩展 */
@media (min-width: 768px) {
  .card { padding: 20px; }
}
```

### 3. 渐进增强

- 先实现核心功能
- 再添加动画和交互
- 最后优化细节体验

### 4. 代码复审要点

- 响应式数据是否正确使用 `reactive` / `ref`
- 移动端是否使用了正确的视口单位 (`100dvh`)
- 弹层是否使用了 `position: fixed`
- 背景滚动是否正确锁定
- 组件是否正确释放资源 (如 G2 chart destroy)
- 动画是否使用了 `transform` / `opacity`
