# 项目避坑指南

本文档总结仪表盘编辑器项目开发中积累的经验和教训，供其他项目参考借鉴。

## 一、技术选型经验

### 1. Vue 3 + Vite 项目初始化

```bash
npm create vite@latest my-app -- --template vue
```

**经验**：使用 `npm create vite` 比手动配置更规范，避免版本兼容问题。

### 2. 状态管理选型

| 场景 | 推荐方案 |
|------|----------|
| 中小型 Vue 3 项目 | `reactive()` 单例模式 |
| 复杂多组件状态 | Pinia |
| 简单组件内状态 | `ref()` / `reactive()` |

**本项目采用**：在 `src/composables/useEditorState.js` 中使用 `reactive()` 单例，所有组件通过 `useEditorState()` 共享状态。

```javascript
// 正确写法 - 单例模式
const state = reactive({ charts: [], configOpen: false })
export function useEditorState() {
  return state
}

// 错误写法 - 每个组件创建新实例
export function useEditorState() {
  return reactive({ charts: [] }) // ❌ 每次调用创建新响应式对象
}
```

### 3. UI 组件库选择

- **Ant Design Vue 4.x**：功能完整，适合中后台管理系统
- **Element Plus**：文档友好，Vue 3 支持好
- **Naive UI**：主题定制灵活，TypeScript 支持好

**本项目采用**：Ant Design Vue ^4.2.6 + AntV G2 ^5.4.8

## 二、移动端开发避坑

### 1. 视口高度单位

| 单位 | 说明 | 移动端表现 |
|------|------|----------|
| `100vh` | 视口高度 | ❌ 包含浏览器工具栏，导致计算偏差 |
| `100dvh` | 动态视口高度 | ✅ 自动适配工具栏（推荐） |
| `100svh` | 最小动态视口高度 | ✅ 兼容性好 |

```css
/* 正确 */
.canvas {
  height: 100dvh;
}

/* 错误 */
.canvas {
  height: 100vh; /* ❌ 移动端会出现滚动问题 */
}
```

### 2. 弹层面板定位

```css
/* 正确 */
.config-panel {
  position: fixed;  /* ✅ 相对于视口定位 */
  top: 0;
  right: 0;
  bottom: 0;
}

/* 错误 */
.config-panel {
  position: absolute;  /* ❌ 相对于父容器定位，易出问题 */
}
```

### 3. 移动端滚动穿透

**问题**：打开弹层后，底层页面仍可滚动。

**解决方案**：

```css
/* 锁定背景滚动 */
body.noscroll {
  overflow: hidden;
  position: fixed;
  width: 100%;
  height: 100%;
}
```

```javascript
// 弹层打开时锁定
watch(() => state.configOpen, (open) => {
  if (open && isMobile.value) {
    document.body.classList.add('noscroll')
  } else {
    document.body.classList.remove('noscroll')
  }
})
```

### 4. 触摸滚动优化

```css
/* 确保触摸事件正确传递 */
.scrollable {
  touch-action: pan-y;  /* 只允许垂直滚动 */
  -webkit-overflow-scrolling: touch;  /* iOS 惯性滚动 */
}
```

### 5. overlay 触摸穿透

```css
/* overlay 阻挡了底层滚动时，让触摸穿透 */
.config-overlay.visible {
  pointer-events: none;  /* 允许触摸穿透到底层 */
}
```

## 三、Vue 3 开发经验

### 1. Composition API 最佳实践

```javascript
// 正确 - 组合式函数
import { ref, computed, watch } from 'vue'

export function useChartManager() {
  const charts = ref([])
  const activeChartId = ref(null)

  const activeChart = computed(() =>
    charts.value.find(c => c.id === activeChartId.value)
  )

  function addChart(config) {
    charts.value.push({ id: Date.now(), ...config })
  }

  return { charts, activeChart, activeChartId, addChart }
}

// 使用
const { charts, addChart } = useChartManager()
```

### 2. 模板语法迁移（从 jQuery/Vue 2 迁移）

| jQuery / Vue 2 | Vue 3 Composition |
|---------------|-------------------|
| `getElementById('id')` | `v-model` 双向绑定 |
| `innerHTML` | `v-html` 或文本插值 |
| `onclick` | `@click` |
| `getAttribute('data-xxx')` | 传递 props 或用 `ref` |

```html
<!-- Vue 2 / 原生 -->
<div id="chart-1" onclick="handleClick(this)" data-id="1"></div>

<!-- Vue 3 -->
<div :data-chart-id="chart.id" @click="handleClick(chart.id)"></div>
```

### 3. 响应式数据注意事项

```javascript
// 正确 - 深层响应
const state = reactive({
  charts: [{ title: 'CPU', thresholds: [] }]
})
state.charts[0].title = '内存'  // ✅ 触发更新

// 正确 - 使用 push 修改数组
state.charts.push(newChart)

// 错误 - 直接替换数组
state.charts = newCharts  // ❌ 如果不是新对象引用，可能不触发更新

// 正确 - 使用解包后的 ref
const count = ref(0)
function increment() {
  count.value++  // 需要 .value
}
```

## 四、图表开发经验

### 1. SVG 图表渲染

```javascript
// SVG 图表使用 v-html 渲染
function renderChartSVG(type, color) {
  const svgMap = {
    numeric: `<svg>...</svg>`,
    line: `<svg>...</svg>`,
    // ...
  }
  return svgMap[type] || ''
}
```

```html
<!-- 模板中使用 -->
<div v-html="renderChartSVG(chart.type, chart.color)"></div>
```

### 2. 图表推荐逻辑

```
| 周期     | 场景               | 推荐图表               |
|---------|-------------------|---------------------|
| 当前值   | 指定单个资源        | 数值卡片 → 折线图     |
| 当前值   | 其他 (TopN/分组)   | 柱状图 → 数值卡片     |
| 时间段   | 任意               | 折线图 → 面积图       |
```

**注意**：
- 推荐列表只显示推荐的图表类型
- 用户可以不使用推荐，选择其他图表
- 当前值模式推荐数值图/柱状图，趋势模式推荐折线图/面积图

### 3. 阈值管理

```javascript
// 阈值更新函数
function updateThValue(chartId, index, value) {
  const chart = state.charts.find(c => c.id === chartId)
  if (chart && chart.thresholds[index]) {
    chart.thresholds[index].value = value
  }
}

function updateThLevel(chartId, index, level) {
  const chart = state.charts.find(c => c.id === chartId)
  if (chart && chart.thresholds[index]) {
    chart.thresholds[index].level = level
  }
}
```

## 五、级联数据结构设计

### 1. 三级级联结构

```javascript
const CASCADE = {
  group: [
    {
      name: '分类名称',
      items: [
        {
          name: '子数据集名称',
          metrics: ['指标1', '指标2', ...]  // 第三级：指标列表
        }
      ]
    }
  ],
  resource: [/* 类似结构 */]
}
```

### 2. 级联选择器逻辑

```javascript
// 获取级联数据
function getCascadeOptions(type) {
  return CASCADE[type]?.map(group => ({
    label: group.name,
    value: group.name,
    children: group.items.map(item => ({
      label: item.name,
      value: item.name
    }))
  })) || []
}

// 获取指标列表
function getMetricsByCategory(category, subItem, type = 'group') {
  const group = CASCADE[type]?.find(g => g.name === category)
  if (!group) return []
  const item = group.items.find(i => i.name === subItem)
  return item?.metrics || []
}
```

## 六、常见问题排查

### 1. 样式不生效

- 检查是否使用了 `<style scoped>`， scoped 样式需要带 data 属性
- 检查 CSS 选择器优先级
- 检查是否有组件层级覆盖问题

### 2. 响应式数据不更新

```javascript
// 检查是否直接修改了对象属性而不是响应式对象
const obj = reactive({ a: 1 })
obj.a = 2  // ✅ 正确

const plain = { a: 1 }
plain.a = 2  // ❌ 非响应式

// 检查是否使用了非响应式的解构
const { a } = state  // ❌ 失去响应式
const { a } = toRefs(state)  // ✅ 保持响应式
```

### 3. 移动端点击延迟

```css
/* 移除点击延迟 */
* {
  touch-action: manipulation;
}
```

### 4. iOS safari 兼容性问题

- 避免使用 `100vh`，使用 `100dvh` 或 `100svh`
- 使用 `-webkit-` 前缀兼容 CSS 属性
- 慎用 `position: sticky`，在 iOS 有时表现异常
