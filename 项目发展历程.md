## 三、开发历程（从 0 到现在的阶段演进）

### Phase 0：初始项目创建（2026-05-14）

**起点**：一个单纯的 Dashboard 编辑器，只有图表网格 + 配置面板。

```bash
npm create vite@latest dashboard-editor -- --template vue
cd dashboard-editor
npm install
npm install ant-design-vue@4 @antv/g2 @fortawesome/fontawesome-free
```

**初始文件**：
- `src/main.js` — Vue + AntD 全量注册
- `src/App.vue` — 主布局 + CSS 变量（华为 HarmonyOS 浅色主题）
- `src/composables/useEditorState.js` — 状态管理核心
- `src/components/ChartGrid.vue` — 图表网格
- `src/components/ConfigPanel.vue` — 数据/样式配置面板
- `src/components/ChartRenderer.vue` — G2 图表渲染
- `src/components/MultiSelect.vue` — 多选下拉组件

**初始状态**：只有 4 个 Dashboard（虚拟机/容器/开发环境/测试环境），每个 Dashboard 默认 10 个预置图表。

### Phase 1：图表编辑器完善（2026-05-14 ~ 05-15）

**里程碑**：
- 级联选择器（分类 → 子数据集 → 指标）
- 图表推荐逻辑（根据指标类型 + 时间范围智能推荐）
- 阈值配置（紧急/重要/次要/提示四级）
- 移动端适配（`100dvh`、弹层定位、滚动优化）

**关键决策**：
- 不用 Pinia/Vuex，用 `reactive()` 单例 composable
- CSS 变量定义主题，`--brand: #007DFF` 蓝色主题

### Phase 2：运维中心系统扩展（2026-05-17 ~ 06-17）

**里程碑**：

### 4.2 状态管理 — reactive 单例模式

```javascript
// src/composables/useEditorState.js — 核心 603 行

// 状态只创建一次（模块级单例）
const state = reactive({
  dashboards: [...],
  currentDashboardId: 1,
  editMode: false,
  charts: [...],
  selectedId: null,
  configOpen: false,
  // ...
})

// 所有组件共享同一个 state
export function useEditorState() {
  return { state, currentChart, addChart, delChart, ... }
}
```

**关键经验**：
- `reactive()` 必须在模块顶层创建，不能在函数内返回新对象
- 通过 `computed` 派生状态（`currentChart`、`recommendedCharts` 等）
- 所有方法直接修改 `state` 的属性，保持响应式

## 七、移动端适配经验

### 7.1 关键问题与解决方案

| 问题 | 方案 |
|------|------|
| 移动端视口高度偏差 | `100dvh` 替代 `100vh` |
| 弹层定位不准 | `position: fixed` 替代 `absolute` |
| 背景滚动穿透 | `body.noscroll { overflow: hidden; position: fixed }` |
| iOS 惯性滚动 | `-webkit-overflow-scrolling: touch` |
| 触摸延迟 | `touch-action: manipulation` |
| 移动端按钮被压缩 | `flex-shrink: 0` 锁定 |

### 7.2 断点策略

| 断点 | 说明 |
|------|------|
| `> 1024px` | 桌面端完整布局 |
| `768px - 1024px` | 平板，侧滑面板宽度缩小 |
| `420px - 768px` | 手机竖屏，面板全屏 |
| `< 420px` | 小屏手机，进一步缩小间距 |

## 八、开发约定与避坑

### 8.1 AGENTS.md 中的关键约定

- 导航栏 `height: 48px`，`#app` 有 `padding-top: 48px`
- 嵌套路由父容器：左侧 200px 导航 + `<router-view />`
- CSS 变量在 `App.vue:272-285`
- 自定义侧滑面板：`position: fixed` + 右滑动画
- 列表页模板：`.page-header > h3` → `.filter-bar` → `a-table`
- 控件默认 32px（不加 `size="small"`）
- 间距：header→bar=16px, bar→table=16px, gap=8px

### 8.2 常见陷阱

| 陷阱 | 解决方案 |
|------|---------|
| HMR 静默失效 | 刷新页面 |
| `a-table` `:row-class-name` 签名 | 用 `(record, index)` 而非 `({ record })` |
| template 箭头函数绑定 | AntD 回调用 `function() {}` |
| G2 `chart.render()` | 是 async，需 await |
| G6 v5 无 `sourceAnchor` | 用 port 系统 |
| fixed 元素被裁剪 | 挂 body 层级，不在 `overflow:hidden` 内 |
| `reactive()` 每次返回新对象 | 状态必须在模块顶层创建 |
| 直接替换整个数组 | 用 `push`/`splice` 修改，不直接赋值 |

## 十、提示词策略（AI 辅助开发）

### 10.1 核心工作流

整个项目使用 AI Agent（Claude Code / Opencode）辅助开发，主要策略：

1. **先文档后代码**：复杂功能先写设计文档（`Dashboard产品功能规划.md`、`运维中心系统设计.md`），再让 AI 实现
2. **经验沉淀**：每个阶段的问题和解决方案写入 `document/` 和 `项目避坑指南.md`
3. **迭代式开发**：小步快跑，每次提交聚焦一个功能点
4. **文档驱动**：`AGENTS.md` 持续更新，沉淀项目约定供 AI 读取

### 10.2 提示词文件

项目中维护了一个核心提示词文件 `待办-dashboard.txt`（1200+ 行），记录了所有开发过程中的指令、样式约定和页面规范。

### 10.3 典型提示词模式

#### 通用约束（文件头部）

```
【单独使用】：更新到github
【回退】：回退到上个git版本
【不要直接修改】先不修改，说一下你的理解
【压缩上下文，60%左右时】/compact
【图表拓扑】涉及图表使用ant G2，涉及拓扑使用ant G6，严禁使用svg直接绘制
```

这 5 条是贯穿全程的"宪法级"约束：
- `【不要直接修改】` — 所有样式/布局调整先让 AI 复述理解，确认无误后再执行，避免误改
- `【图表拓扑】` — 强制使用 G2/G6 渲染图表，禁止手写 SVG
- `【压缩上下文】` — 上下文接近 60% 时触发 compact，保持 AI 注意力集中

#### 页面布局约定

```
所有涉及搜索、列表、按钮的页面都要调整。
"第1行  按钮（如有），独占一行，右对齐
第2行  搜索+筛选控件，独占一行，搜索框拉宽至和列表右边缘对齐（用 flex: 1）
第3行  表格"
```

这条约定覆盖了 50+ 个列表页的统一结构，确保全站视觉一致性。

#### 间距约定

```
alarm-content的padding是24px；page-view的padding也是24px，
加起来就48px了，建议把告警管理下所有页面的page-view的padding的左右都改为0。
```

#### 组件尺寸约定

```
按钮大小太小，使用默认按钮大小高度32px，按钮内文字左右间距16px。
按钮直接间距8px
```

#### 页面详细描述提示词

对于复杂页面，使用结构化描述 + ASCII 布局 + 表格，例如告警管理页面：

```

### 13.2 五大宪法级约束（文件头部）

```

这 5 条约束贯穿整个开发周期，是项目质量的保障：

| 约束 | 作用 | 效果 |

| `【回退】` | 快速回滚到上一版本 | 试错成本低 |
| `【单独使用】` | 单独执行 git push | 避免意外提交 |
| `【压缩上下文】` | 上下文 60% 时触发 compact | 保持 AI 注意力集中 |
