# Dashboard 产品功能规划

本文档规划监控仪表盘产品的完整功能，包含当前已实现功能和待开发功能。

---

## 一、产品概述

### 1.1 产品定位

面向运维人员、业务人员的数据可视化监控平台，支持多环境、多区域的资源监控仪表盘创建、编辑和查看。

### 1.2 范围约定

| 功能 | 状态 | 说明 |
|------|------|------|
| Dashboard 存储 | 暂不做 | 当前仅内存状态，重启丢失 |
| 权限控制 | 暂不做 | 不考虑多角色权限 |
| 模板市场 | 暂不做 | 后续版本考虑 |
| 告警集成 | 暂不做 | 后续版本考虑 |
| 数据互通 | 不支持 | 每个 Dashboard 独立配置 Region 和时间范围 |

### 1.3 Region 设计

- **数据源**：前端硬编码示意（后续从后端 API 获取）
- **独立性**：每个 Dashboard 独立保存自己的 Region 配置，切换 Dashboard 时不联动

```javascript
// Region 数据源示例

  { id: 'cn-north-3', name: '华北区域三', code: 'cn-north-3' },
```

## 二、状态管理

### 2.1 两种状态模式

```
┌─────────────────────────────────────────────────────────┐
│  查看态 (默认)                                          │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 左侧: Dashboard页签  │  右侧: Region | 编辑    │   │
│  └─────────────────────────────────────────────────┘   │
│  - 图表点击 → 无动作（不打开侧滑编辑）                  │
│  - 图表拖拽 → 禁用                                     │
│  - 筛选/刷新 → 启用                                    │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│  编辑态                                                │
│  ┌─────────────────────────────────────────────────┐   │
│  │ 左侧: Dashboard页签  │  Region | 保存 | 退出    │   │
│  └─────────────────────────────────────────────────┘   │
│  - 图表点击 → 打开侧滑编辑面板                         │
│  - 图表拖拽 → 启用（网格布局可调换位置）                │
│  - 布局调整 → 启用                                     │
└─────────────────────────────────────────────────────────┘
```

### 2.2 切换交互

| 操作 | 触发 | 结果 |
|------|------|------|
| 进入编辑 | 查看态点击"编辑"按钮 | 切换为编辑态，右上角显示"保存"+"退出编辑" |
| 保存 | 编辑态点击"保存"按钮 | 提示"保存成功"，保持编辑态 |
| 退出编辑 | 编辑态点击"退出编辑"按钮 | 放弃未保存修改，切换回查看态 |

## 四、编辑态功能

### 4.1 网格布局与拖拽（待开发）

#### 4.1.1 网格系统

- **布局**：使用 CSS Grid 实现响应式网格
- **列数**：自动根据容器宽度调整（1-4列）
- **行高**：图表高度统一，可配置

```
// 网格布局示意
.chart-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(360px, 1fr));
  grid-auto-rows: minmax(280px, 1fr);
  gap: 16px;
}
```

#### 4.1.2 拖拽排序

- **方式**：拖拽图表卡片调整位置
- **视觉反馈**：
  - 拖拽时图表半透明，显示占位框
  - 目标位置高亮提示
  - 释放后自动吸附到新位置
- **辅助拖拽手柄**：图表标题左侧显示拖拽图标，鼠标悬停时显示
- **保存**：拖拽完成后自动保存位置

#### 4.1.3 拖拽交互细节

| 场景 | 行为 |
|------|------|
| 开始拖拽 | 图表变为半透明，显示在原位置显示占位虚线框 |
| 拖拽中 | 实时计算插入位置，目标位置高亮边框 |
| 释放 | 图表动画移动到新位置，更新排序数据 |

#### 4.1.4 图表编辑

- **触发**：点击图表卡片（非拖拽区域）
- **面板**：右侧滑出配置面板

#### 4.1.5 图表复制/删除

| 操作 | 行为 |
|------|------|
| 复制 | 生成副本，标题加"(副本)"，添加到画布末尾 |
| 删除 | 显示确认对话框，删除后自动选中相邻图表 |

### 4.2 配置面板（已实现）

#### 4.2.1 数据配置 Tab

| 区块 | 字段 | 说明 |
|------|------|------|
| 数据集 | 类型选择 | 按分组 / 按资源类型 |
| | 级联选择 | 三级：分类 → 子数据集 → 指标 |
| 统计周期 | 时间范围 | 实时当前值 / 最近1h/6h/24h/7d/30d |
| 采集周期 | 间隔 | 10s / 30s / 1m / 5m / 15m / 1h |
| 监控指标 | 多选 | 从数据集筛选的指标列表 |
| 监控对象 | 对象类型 | 全部资源 / 资源分组 / 指定资源 |
| | 聚合方式 | 平均值 / 最大值 / 最小值 |
| | TopN 数量 | 数字微调器 |
| 推荐图表 | 自动推荐 | 根据指标和时间范围推荐图表类型 |

#### 4.2.2 样式配置 Tab

| 区块 | 字段 | 说明 |
|------|------|------|
| 基础信息 | 图表名称 | 输入框 |

| | 备注 | 多行文本 |
| 图例 | 位置 | 不显示 / 顶部 / 底部 / 右侧 |
| 阈值 | 阈值行 | 动态添加：数值 + 等级（紧急/重要/次要/提示） |
| 链接 | 开启开关 | Toggle |
| | 链接 URL | 输入框，点击标题旁图标跳转 |

### 4.3 添加图表

## 九、技术实现要点

### 9.1 拖拽排序方案

可选用方案：

| 方案 | 优点 | 缺点 |
|------|------|------|
| HTML5 Drag and Drop | 原生 API，无需依赖 | 移动端支持差 |
| Sortable.js | 成熟稳定，移动端支持好 | 需引入额外库 |
| vuedraggable (Vue 封装) | Vue 集成好 | 依赖 Vue.Draggable |
| 自定义实现 | 无依赖，可定制 | 开发成本高 |

**建议**：使用 `vuedraggable` 或 `Sortable.js`，兼容桌面和移动端

### 9.2 Grid 布局要点

```css
.chart-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(360px, 1fr));
  grid-auto-rows: minmax(280px, 1fr);
  gap: 16px;
}

/* 响应式断点 */
@media (max-width: 1024px) {
  .chart-grid { grid-template-columns: repeat(auto-fit, minmax(280px, 1fr)); }
}
@media (max-width: 640px) {
  .chart-grid { grid-template-columns: 1fr; }
}
```
