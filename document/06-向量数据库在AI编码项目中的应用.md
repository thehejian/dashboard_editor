## 二、向量数据库简介

向量数据库是专门用于存储和检索 **向量 Embedding** 的数据库，核心能力：
- 高效的向量相似度搜索
- 支持高维数据
- 适用于 AI/机器学习场景

**pgvector** 是 PostgreSQL 的向量扩展，支持：
- 向量数据类型
- IVFFlat 索引（近似最近邻搜索）
- HNSW 索引（更快的近似搜索）

## 三、在 AI 编码项目中能做的事

### 1. 代码/文档语义搜索

#### 1.1 自然语言搜代码

**场景**：用自然语言描述需求（如"登录表单组件"），搜出匹配的代码片段

**实现方案**：

```
用户输入 → Embedding → 向量搜索 → 返回代码片段
```

**表结构设计**：

```sql
CREATE TABLE code_snippets (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255),           -- 标题，如"登录表单"
  description TEXT,             -- 描述，如"Vue3登录表单组件"
  code_content TEXT,            -- 代码内容
  language VARCHAR(50),         -- 语言，如 vue, js, css
  embedding vector(1536),        -- 向量（OpenAI text-embedding-3-small 是 1536 维）
  metadata JSONB,               -- 扩展信息，如 { "author": "张三", "tags": ["表单", "登录"] }
  created_at TIMESTAMP DEFAULT NOW()
);

-- 创建向量索引（HNSW 索引，搜索更快）
CREATE INDEX ON code_snippets USING hnsw (embedding vector_cosine_ops);
```

**调用流程**：

```javascript
// 1. 用户输入描述
const query = "登录表单组件"

// 2. 调用 Embedding API 获取向量
const embedding = await getEmbedding(query)  // 返回 1536 维向量

// 3. 向量搜索
const result = await pool.query(`
  SELECT title, code_content, language,
         1 - (embedding <=> $1::vector) as similarity
  FROM code_snippets
  WHERE 1 - (embedding <=> $1::vector) > 0.7
  ORDER BY embedding <=> $1::vector
  LIMIT 5
`, [embedding])
```

#### 1.2 代码片段推荐

**场景**：根据当前编辑内容（如输入了 `<a-input`），推荐常用代码模板

**实现方案**：

```
用户输入 → 提取关键词 → 向量搜索 → 返回代码模板
```

**表结构设计**：

```sql
CREATE TABLE code_templates (
  id SERIAL PRIMARY KEY,
  name VARCHAR(255),             -- 模板名称，如"输入框"
  trigger_keyword VARCHAR(100), -- 触发关键词，如 a-input, input
  template_content TEXT,         -- 模板内容
  category VARCHAR(50),         -- 分类，如"表单组件"
  embedding vector(1536),        -- 向量
  usage_count INT DEFAULT 0,    -- 使用次数，用于排序
  created_at TIMESTAMP DEFAULT NOW()
);
```

**调用流程**：

```javascript
// 1. 用户输入触发
const input = "<a-input"

// 2. 提取关键特征
const features = extractFeatures(input)  // 可能包含"输入框", "表单"等

// 3. 获取向量并搜索
const embedding = await getEmbedding(features)
const result = await pool.query(`
  SELECT name, template_content, usage_count
  FROM code_templates
  ORDER BY embedding <=> $1::vector, usage_count DESC
  LIMIT 3
`, [embedding])
```

#### 1.3 文档问答（RAG）

**场景**：基于项目文档的自然语言问答，如"运维手册怎么操作？"

**实现方案**：

```
用户问题 → 向量化 → 相似度搜索 → 上下文 + LLM 生成回答
```

**表结构设计**：

```sql
CREATE TABLE docs (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255),           -- 文档标题
  content TEXT,                 -- 文档内容（可以很长）
  chunk_text TEXT,              -- 分片后的文本块
  chunk_index INT,              -- 分片索引
  embedding vector(1536),       -- 该分片的向量
  doc_type VARCHAR(50),         -- 文档类型，如"运维手册", "API文档"
  metadata JSONB,               -- 扩展信息
  created_at TIMESTAMP DEFAULT NOW()
);

-- 为文档添加全文索引
CREATE INDEX ON docs USING gin (to_tsvector('zh', chunk_text));
```

**调用流程**：

```javascript
// 1. 用户提问
const question = "如何创建新的监控图表？"

// 2. 获取问题向量
const questionEmbedding = await getEmbedding(question)

// 3. 检索相关文档块
const docs = await pool.query(`
  SELECT title, chunk_text,
         1 - (embedding <=> $1::vector) as similarity
  FROM docs
  WHERE 1 - (embedding <=> $1::vector) > 0.75
  ORDER BY embedding <=> $1::vector
  LIMIT 3
`, [questionEmbedding])

// 4. 拼接到 LLM 上下文
const context = docs.rows.map(d => d.chunk_text).join('\n\n')
const prompt = `根据以下文档回答问题：\n\n${context}\n\n问题：${question}`

// 5. 调用 LLM 生成回答
const answer = await callLLM(prompt)
```

## 四、优势分析

### 对比其他方案

| 特性 | 向量数据库 (pgvector) | 普通数据库 | Elasticsearch |
|------|---------------------|------------|---------------|
| 向量存储 | ✅ 原生支持 | ❌ 需要 JSON | ✅ 支持 |
| 相似度搜索 | ✅ 高效 | ❌ 全文匹配 | ✅ 支持 |
| 混合查询 | ✅ SQL 联合 | ❌ 复杂 | ⚠️ 有限 |
| 部署成本 | ✅ 低（PostgreSQL） | - | 较高 |
| 生态集成 | ✅ 丰富 | - | 丰富 |

### 在本项目中的优势

1. **一体化**：复用现有 PostgreSQL，无需额外服务
2. **SQL 兼容**：用熟悉的 SQL 查询
3. **AI 友好**：无缝对接主流 Embedding 模型
4. **轻量**：pgvector 扩展仅 ~1MB
