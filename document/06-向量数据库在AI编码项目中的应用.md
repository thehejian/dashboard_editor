## 二、向量数据库简介

向量数据库是专门用于存储和检索 **向量 Embedding** 的数据库，核心能力：
- 高效的向量相似度搜索
- 支持高维数据
- 适用于 AI/机器学习场景

**pgvector** 是 PostgreSQL 的向量扩展，支持：
- 向量数据类型
- IVFFlat 索引（近似最近邻搜索）
- HNSW 索引（更快的近似搜索）

## 四、优势分析

### 对比其他方案

| 特性 | 向量数据库 (pgvector) | 普通数据库 | Elasticsearch |
|------|---------------------|------------|---------------|
| 向量存储 | ✅ 原生支持 | ❌ 需要 JSON | ✅ 支持 |
| 相似度搜索 | ✅ 高效 | ❌ 全文匹配 | ✅ 支持 |
| 混合查询 | ✅ SQL 联合 | ❌ 复杂 | ⚠️ 有限 |
| 部署成本 | ✅ 低（PostgreSQL） | - | 较高 |
| 生态集成 | ✅ 丰富 | - | 丰富 |

### 在本项目中的优势

1. **一体化**：复用现有 PostgreSQL，无需额外服务
2. **SQL 兼容**：用熟悉的 SQL 查询
3. **AI 友好**：无缝对接主流 Embedding 模型
4. **轻量**：pgvector 扩展仅 ~1MB
