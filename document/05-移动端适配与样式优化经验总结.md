# 移动端适配与样式优化经验总结

> 更新日期：2025-05-17
> 项目：运维中心 (Vue 3 + Vite + Ant Design Vue)

## 一、问题处理经验

### 1. 首页滚动问题

#### 问题描述
- 1k 屏：首页和仪表盘都无法滚动
- 手机端：仪表盘可以滚动，首页不行

#### 根因分析
1. `.main` 容器没有 explicit height，`flex: 1` 在某些分辨率下不生效
2. `router-view` 作为滚动容器不稳定，需要包一层 div
3. Dashboard 移动端有 explicit height (`calc(100dvh - 48px)`)，首页没有

#### 解决步骤
1. 回滚到 git 正常状态
2. 给 router-view 包一层 div 作为滚动容器：
   ```html
   <div v-else class="page-view-container">
     <router-view />
   </div>
   ```
3. 添加 CSS：
   ```css
   .main { height: calc(100vh - 48px); }
   .page-view-container { height: 100%; overflow-y: auto; -webkit-overflow-scrolling: touch; }
   ```
4. 移动端媒体查询添加：
   ```css
   @media (max-width: 768px) {
     .main { height: calc(100dvh - 48px); }
     .page-view-container { height: 100%; }
   }
   ```

#### 教训
- 优先回滚再修改，避免改出更多问题
- flex 子元素需要父容器有 explicit height
- 移动端用 `100dvh` 替代 `100vh` 解决底部安全区域

### 2. 首页卡片"查看全部"按钮渲染问题

#### 问题描述
按钮显示原始 render 函数代码：
```
h('a-button', {type: 'link', onClick: () => _ctx.$router.push('/alarm/realtime')}, ['查看全部 '])
```

#### 根因
- `:extra` prop 直接传入 render 函数，Vue 模板无法解析

#### 解决
```html
<!-- 错误写法 -->
<a-card :extra="h => h('a-button', ...)">

<!-- 正确写法 -->
<a-card>
  <template #extra>
    <a-button type="link" @click="$router.push('/alarm/realtime')">查看全部</a-button>
  </template>
</a-card>
```

### 3. 顶部导航图标适配

#### 需求
- 网页端：icon + 文字
- 手机端：仅显示 icon

#### 实现
1. 给每个模块添加 icon：
   ```html
   <router-link to="/" class="nav-item">
     <i class="fa-solid fa-house"></i><span>首页</span>
   </router-link>
   ```

2. 添加 CSS：
   ```css
   .module-nav .nav-item { display: flex; align-items: center; gap: 6px; }
   .module-nav .nav-item i { font-size: 14px; }
   ```

3. 移动端隐藏文字：
   ```css
   @media (max-width: 768px) {
     .module-nav .nav-item span { display: none; }
   }
   ```

### 4. 仪表盘移动端布局优化

#### 需求
- 区域左对齐，四个 icon 按钮整体右对齐
- 间距 8px
- 网页端和移动端分开处理

#### 实现
1. 添加移动端下拉选择器：
   ```html
   <div class="dashboard-tabs-mobile">
     <a-select v-model="state.currentDashboardId">
       <a-select-option v-for="db in dashboards" :value="db.id">{{ db.title }}</a-select-option>
     </a-select>
   </div>
   ```

2. 包裹 icon 按钮：
   ```html
   <div class="dashboard-action-icons">
     <!-- 四个 icon 按钮 -->
   </div>
   ```

3. CSS：
   ```css
   .dashboard-action-icons { display: flex; gap: 8px; margin-left: auto; }
   ```

4. 移动端媒体查询：
   ```css
   @media (max-width: 768px) {
     .dashboard-tabs { display: none; }
     .dashboard-tabs-mobile { display: block; width: 100%; }
     .dashboard-actions { justify-content: space-between; }
   }
   ```

## 二、避坑指南

### CSS 布局
1. **flex height 传递**：flex 子元素需要父容器有 explicit height
2. **router-view 不稳定**：包一层 div 比直接用 router-view 更可靠
3. **移动端用 100dvh**：避免底部安全区域问题

### Vue 组件
1. **render 函数**：不要在 template 中写 render 函数，用 slot 替代
2. **Ant Design Vue**：优先用 slot，prop 可能不支持复杂写法

### 响应式设计
1. **渐进式适配**：先确保桌面端，再做移动端
2. **分而治之**：网页端和移动端完全分离，用不同结构
3. **媒体查询顺序**：从小到大，避免样式覆盖问题

## 三、CSS 技巧总结

### 滚动容器
```css
.page-view-container {
  overflow-y: auto;
  overflow-x: hidden;
  -webkit-overflow-scrolling: touch;  /* iOS 滚动 */
  touch-action: pan-y;                 /* 移动端触摸 */
}
```

### 移动端高度
```css
.main {
  height: calc(100vh - 48px);        /* 桌面端 */
}

@media (max-width: 768px) {
  .main {
    height: calc(100dvh - 48px);     /* 移动端，适配安全区域 */
  }
}
```

### Flex 布局
```css
.dashboard-actions {
  display: flex;
  justify-content: space-between;   /* 两端对齐 */
}

.dashboard-action-icons {
  margin-left: auto;                 /* 右侧对齐 */
  gap: 8px;                         /* 间距 */
}
```

## 四、调试技巧

### 排查滚动问题
1. 检查父容器是否有 explicit height
2. 检查 overflow 是否正确设置
3. 检查 flex 子元素是否有 min-height: 0
4. 移动端检查 -webkit-overflow-scrolling

### 排查样式不生效
1. 检查 CSS specificity
2. 检查媒体查询顺序
3. 检查是否有其他样式覆盖
4. 用浏览器开发者工具验证
