# 移动端弹层面板问题修复经验

## 问题描述

手机端打开配置面板时：
- 侧滑面板无法滚动
- 底部取消/更新按钮被挤出视口

## 根因分析

1. 使用 `position: absolute` 定位面板，高度计算依赖父容器，在复杂 flex 布局中容易出错
2. 使用 `100vh` 单位，移动端浏览器动态工具栏导致视口高度计算偏差

## 解决方案

### 1. 面板定位
```css
.config-panel {
  position: fixed; /* 而非 absolute */
  top: 0;
  right: 0;
  bottom: 0;
  display: flex;
  flex-direction: column;
}
```

### 2. 视口高度
```css
@media (max-width: 768px) {
  .config-panel {
    height: 100svh; /* 动态视口高度，自动适配移动端工具栏 */
  }
}
```

### 3. 锁定背景滚动
```css
body.noscroll {
  overflow: hidden;
  position: fixed;
  width: 100%;
  height: 100%;
}
```
```javascript
watch(() => state.configOpen, (open) => {
  if (isMobile.value) {
    if (open) document.body.classList.add('noscroll')
    else document.body.classList.remove('noscroll')
  }
})
```

### 4. 触摸滚动支持
```css
.config-scroll {
  touch-action: pan-y;
  -webkit-overflow-scrolling: touch;
}
```

## 修改的文件

- `src/components/ConfigPanel.vue` - CSS 样式修复
- `src/App.vue` - 添加移动端检测和 body 滚动锁定

## 关键经验

| 问题 | 错误方案 | 正确方案 |
|------|----------|----------|
| 弹层定位 | `position: absolute` | `position: fixed` |
| 视口高度 | `100vh` | `100svh` / `100dvh` |
| 滚动穿透 | 忽略 | `body { overflow: hidden }` |
| 触摸滚动 | 仅 `overflow: auto` | 加上 `touch-action: pan-y` |

## 调试技巧

1. 先用 `position: fixed` 快速验证问题是否与定位相关
2. 简化样式，确认基本功能正常后再加细节
3. 移动端调试时注意浏览器工具栏对视口的影响
