# 经验教训总结

## 一、移动端滚动问题

### 问题描述
手机端打开配置面板时，主页面图表区域无法滚动。

### 根因分析
1. `.config-overlay` 设置了 `pointer-events: auto`，阻挡了触摸事件到底层的滚动区域
2. 画板高度没有固定，内容溢出后被父容器裁剪

### 解决方案
1. overlay 的 `pointer-events` 改为 `none`，让触摸穿透到底层
2. 手机端画板高度固定为 `calc(100dvh - header高度)`
3. 滚动容器设置正确的高度和 `overflow-y: auto`
4. 子元素添加 `touch-action: pan-y` 确保触摸事件正确传递

### 关键代码
```css
/* overlay 穿透触摸 */
.config-overlay.visible { pointer-events: none; }

/* 画板高度固定 */
@media (max-width: 768px) {
  .canvas { height: calc(100dvh - 48px); }
  .canvas-scroll { flex: 1; min-height: 0; overflow-y: auto; }
}

/* 子元素触摸传递 */
.chart-card, .chart-body { touch-action: pan-y; }
```

## 二、图表推荐逻辑问题

### 问题描述
1. 当前值场景推荐了面积图，不符合预期
2. 推荐列表显示了一些不应该出现的图表类型

### 根因分析
1. 代码逻辑与需求文档不一致（使用了旧的复杂逻辑）
2. 当前图表类型不在推荐列表中时，仍然显示并高亮
3. 推荐列表过滤逻辑有误

### 解决方案
1. 简化推荐逻辑：
   - 当前值 + 单资源 → 数值卡片
   - 当前值 + 其他 → 柱状图
   - 时间段 → 折线图/面积图
2. 只显示推荐的图表类型（过滤掉不推荐的）
3. 当前选中状态只在推荐列表中生效

### 推荐逻辑最终版

| 周期 | 场景 | 推荐图表 |
|------|------|---------|
| 当前值 | 指定单个资源 | 数值卡片 → 折线图 |
| 当前值 | 其他（TopN/分组/多资源） | 柱状图 → 数值卡片 |
| 时间段 | 任意 | 折线图 → 面积图 |

## 三、经验教训

### 1. 移动端调试
- 使用 `100dvh` / `100svh` 而非 `100vh`，避免浏览器工具栏导致的高度计算错误
- `position: fixed` 比 `position: absolute` 更可靠
- 子元素的 `touch-action` 属性影响父容器滚动

### 2. 逻辑与实现一致性
- 先更新文档再改代码，确保逻辑一致
- 代码修改后要对照文档验证
- 复杂逻辑先简化，再逐步验证

### 3. 推荐功能设计
- 推荐列表只显示推荐类型，不显示全部类型
- 用户可以不使用推荐，选择其他图表
- 不合适的图表显示警告提示
